march 2015 - feb 2016

Joined 18 months after dev started, when major refactoring was required to achieve performance.
  Provided java development and scalability / performance refactoring for a startup focused on fast image, video and audio processing.

  Most challenging task was related to files synchronization between the Orange Cloud and our own Amazon S3 storage.

  Micro services architecture on Amazon EC2 with an ELB in front of each cluster of services. I had full access to servers, both in production and in a small scale replica we used for testing. My work consisted of developing new features (eg: short-links, counters, Orange API integration) and improving performance by benchmarking and redesigning architecture and processes.

   Due to performance issues and high costs concerning the current architecture I have moved communication between micro-services from HTTP to an AMQP software (rabbitmq). I had to remake parts the schema of the database (MongoDB) with several aggregations selected by the way we are using the database and not by respecting some SQL principles as it was before.
